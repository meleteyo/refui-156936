폭 방향 배열(스트리핑)

두께·재질별로 WEB/FU/FL 부재를 폭 방향으로 배열해 광폭 플레이트를 발주 할증률(발주중량÷정미중량) 최소화. 폭 배정도 NP-hard 조합 최적화이므로, 기본은 First-Fit-Decreasing류 그리디 근사 + 로컬서치, "최적" 요구 시 OR-Tools CP-SAT로 승격합니다.

  • 두께·재질 그룹 내 부재를 폭으로 병합 배열(FFD류) → 폭 밴드 안에 채우고, 넘치면 새 플레이트 개시
  • 행위 불변식 발주폭 == Σ(부재폭×배열수) + 에지트림 + 커프×절단선수 (폭 보존) — assert + DB CHECK

  • 확정 CP(is_confirmed)만 수주 확정 허용 — CP 출력이 발주·생산·손익의 단일 진실원

Q3 🔴BH CP 엑셀 양식 구조 — 단일 표인가, 다중 시트·매크로·수식 체인인가? 착수 즉시 제공 가능?

선택지: (A) 단순표 · (B) 다중시트·매크로 · + 제공 가부·시점

영향: 파서 공수·정합성 직결. 양식이 곧 명세이므로 미확보 시 설계 착수 불가 (M3/A2)

Q6 🔴시험성적서 양식 수 × 국가(국내/일본/호주)의 총 종류 수와 각 필드 스키마 확보 가능?

선택지: 국가별 종수 + 샘플 파일

영향: 양식별 개별 스펙(입력폼·검증·PDF 템플릿). 종수 = 2차 공수 승수 (M5/A6)

Q7 🔴"MCS" 정의 — Mill Certificate Sheet인가, Material Control System인가, 사내 약어인가?

선택지: 정의 + 샘플

Q8 🔴더존 ERP/전자세금계산서 연동 범위 — 확장성 DB 설계만인가, 실제 API 연동 개발 포함인가?

Q9 🔴데이터 마이그레이션 범위 — 신규 입력만인가, 과거 엑셀 이력 이관(연수·건수)인가?

Q18 🔴배포·호스팅 주체 — 개발사 관리 클라우드인가, 발주사 온프레미스/자체 클라우드인가?

def optimize_cutting_plan(detail: CuttingPlanDetail) -> CuttingResult:
    """
    두께·재질 그룹의 부재를 폭 방향으로 배열해 광폭 플레이트를 발주.
    할증률(발주중량÷정미중량) 최소화. (폭배정 1D 빈패킹, FFD 휴리스틱)

    후행조건: 모든 부재 배치 · markup_rate >= 1.0
    선행조건(방어적): 폭 밴드 목록 비어있지 않음 · 부재 폭 > 0
                     max(부재폭)+에지 ≤ 밴드 상한 (초과 시 배치불가 예외)
    행위불변식: 각 plate에서 발주폭 == Σ(부재폭×배열수) + 에지트림 + 커프×절단선수 (폭 보존)
    """
    band_max = max(detail.width_bands)         # 발주 가능 최대 폭(예 3300)
    if band_max <= 0:
        raise ValueError("주문 가능한 폭 밴드가 없습니다.")

    strips: list[int] = []
    for m in detail.members:                   # 전개된 부재(폭)
        if m.width_mm <= 0 or m.quantity < 0:
            raise ValueError("부재 폭/수량이 올바르지 않습니다.")
        strips.extend([m.width_mm] * m.quantity)
    if any(w + EDGE_TRIM > band_max for w in strips):
        raise ValueError("부재 폭이 밴드 상한을 초과합니다(배치불가).")

    strips.sort(reverse=True)  # 내림차순(FFD)

    plates: list[list[int]] = []
    room: list[int] = []
    for w in strips:
        placed = False
        for i in range(len(plates)):
            need = w + KERF                     # 스트립 추가 시 커프 가산
            if room[i] >= need:                 # 폭 여유 충분한 첫 플레이트에 배열
                plates[i].append(w); room[i] -= need; placed = True; break
        if not placed:                          # 없으면 새 플레이트 개시(최소폭 1501 보장)
            plates.append([w]); room.append(band_max - w - EDGE_TRIM)

    # ... layouts 생성 · markup = 발주중량합 / 정미중량합
    for l in layouts:                           # 행위불변식 감시: 폭 보존
        assert l.order_width_mm == sum(l.strip_widths) + EDGE_TRIM + KERF * (len(l.strip_widths) - 1)
    return CuttingResult(layouts=layouts, markup_rate=markup,
                         ordered_weight_kg=ordered_kg,
                         solution_type=SolutionType.APPROXIMATE)  # FFD = 근사해
